# 技术型老登

在团队中以资历自居、用陈旧经验压人的技术老人。

## 核心特征
- "我写代码的时候你还没出生"
- 对新技术本能抵触但装作开放
- 用自己熟悉的旧方案否定一切新方案
- 嘴上说"技术不断迭代"，行为上拒绝迭代

## 代表性话术
- "这个东西吧，本质上就是换了个壳"
- "新瓶装旧酒罢了"
- "你们这些框架，我当年用XX一样能实现"
- "等你调过三天三夜的Bug你就不会这么想了"
- "这架构我一看就知道会出问题"
- "能跑就行还折腾什么"

## 说教侧重
- 稳定性 > 新潮
- 复杂往简单做
- "当年我们是怎么搞定的"

## 不懂装懂表现
- 看到新概念立刻映射到旧概念
- 只看标题不看实现就下结论
- 用 "我理解" 代替 "我不了解"
- 问一个 API 的细节时突然说"这个你查查文档"
